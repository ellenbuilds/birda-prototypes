# The Game of Birda — clickable prototypes

Interactive prototypes for the three explored routes. One folder per route so they stay independent.

```
birda-prototypes/
├─ resources/bird-photos/    ← SHARED bird photos (proper cutouts) — used by all routes
└─ game-of-birda/
   ├─ route-2-collections/   ← collections & sets (this one)
   │  ├─ index.html          ← open in a browser
   │  └─ images/
   │     ├─ map-far.jpg
   │     └─ figma-reference/ ← the original P1–P7 Figma exports (reference only)
   ├─ route-1-quests/        ← field guide with quests (feathers, events)
   │  ├─ index.html          ← open in a browser
   │  └─ images/             ← welcome.png (Figma "welcome" frame, node 229:152) + wader-hero.png (cropped from the Figma migration frame)
   └─ route-3-mastery/       ← mastery & identity — "the recognition loop"
      ├─ index.html          ← open in a browser (bird photos pulled from ../../resources/bird-photos/)
      └─ images/figma-reference/welcome.png  ← Figma "Welcome" splash (onboarding screen 1)
```

Bird photos are referenced from the shared `birda-prototypes/resources/bird-photos/` folder (as `../../resources/bird-photos/…`), so every route prototype uses the same source images.

## route-2-collections
A working prototype of a simplified version of our existing onboarding journey that then introduces the **collections & sets** idea. Built as a single self-contained `index.html` (no build step) styled to match production — Rubik, brand tokens (`brand-blue #1F87FE`, `dark-petrol #2D3142`, `light-cloud`), the real screen structures, and real bird photos. Patterns mirror the production app (`/Users/ellen/code/app`, React Native + Expo), reimplemented as plain HTML/CSS/JS for a browser-openable prototype.

Flow (from the Figma "Route 2 · Collecting sets" page). The Figma **P2 "creating your account" loader is intentionally omitted** — it added little:
1. **Welcome** (P1) — flat Figma export (`images/figma-reference/p1.png`); tap to continue
2. **Your birding journey** (P3) — single-select (4 options); button reads *Skip* until you pick
3. **Location** (P4) — “Discover nearby birds”
4. **Start your collection** (P5) — card-stack hero + tap to pick the last bird you saw
5. **A brand-new bird!** (P6) — first-collected celebration
6. **Level up your cards** (P7) — the Garden Birds set (1 of 6); locked slots show a pale bird silhouette + the next target ("Blackbird") or how many more to log to reveal it. Silhouettes live in `images/silhouettes/`. "Find my next bird" → Home.
7. **Home** — the app home screen (map + "65 known species in this area" + Next Targets cards). Built from the Figma export (`images/home.png`); the Eurasian Blue Tit card tag is overlaid to read **"Garden Birds Set"** (instead of its rarity). The subtitle ("3 garden birds to collect") comes straight from the export. The floating **Identify** + **Log** buttons above the bottom sheet are reproduced from the production `ChunkyButton` (rounded, thick coloured bottom edge; Identify = brand-blue, Log = white/inverse). The Identify icon is the app's `IdentifyBirdIcon`; the Log icon is an approximation of the production crow glyph.
8. **What did you see?** (species select) — tapping **Log** on Home opens this species picker (search + list: Unidentified Species, Woodpigeon, Carrion Crow, Robin, Magpie, Blackbird), copied from the `PEX-43-v2` prototype, using the shared cutout photos from `resources/bird-photos/`. The back chevron returns to Home.
9. **Review Sighting** — picking any species opens the review screen (map + pin, photos, notes, sighting details — species/count/date/location/tag — and privacy), also copied from `PEX-43-v2`. The picked species name carries through to the Sighting row. A **"How did you see it?"** selector lets you toggle sighting attributes — Male, Female, Singing, In flight, Bathing (multi-select tiles styled like the card slots). Save / back return to Home or the species list. On arrival a **"How a card works" bottom sheet** slides up (content from the Figma frame *R2 S2 · How a card works*): the card title shows the bird being logged, with its six slots (Male filled, Singing / In flight / Female / Photo / Bathing locked) and "1 of 6 filled — Bronze". Dismiss via **Got it** or the **×**.
10. **Level up (Silver)** — saving the sighting opens a celebration (content from the Figma frame *R2 S10 · Level up (Silver)*): the logged bird's card animates its slots filling one by one, then **levels up Bronze → Silver** — the badge and card border shift to silver, a metallic **silver sheen sweeps diagonally across the panel**, a "SILVER UNLOCKED" pill drops in, and **confetti bursts then falls away**. The card sits in 3D — a soft drop shadow plus a slow, subtle back-and-forth horizontal tilt (respects `prefers-reduced-motion`). **See my collection** → the profile (11).
11. **Profile · Collections** — the app profile screen (header from the Figma *profile header* reference: avatar, name, stats, Edit Profile), with a new **Collections** tab inserted between Summary and Life Lists (active). The tab opens with a banner showing the golden onboarding egg (now **cracked**) and a live **countdown timer** ticking down from 21 hours ("Your golden egg is hatching"). Below, the content reuses the *Level up your cards* design but each card is a **set** rather than a single bird, in three states (all light-blue bordered): **completed** (Coastal visitors — a blue medallion/checkmark badge instead of a bird grid, all dots darker blue, "✓ Complete"), **started** (Garden Birds — member photos + "+3", 3-of-6 light-blue dots), and **not-started** (Winter Waders, Woodland Birds, Birds of Prey, Finches & Buntings — dashed border, silhouette covers). Tabs are sticky; the other tabs' contents aren't prototyped. A bottom tab bar (Explore · Community · **+** · Challenges · Profile) sits at the foot of the screen — its **+** opens the log-a-bird flow (screen 8), as does the Home screen's Log button and its nav **+**. The egg banner is a compact one-liner (small cracked egg + live countdown). The card carries the bronze-card **dot progress indicator** (six circles) under its name, filling in sync with the slots and turning silver on level-up; the count climbs to "4 of 6 filled. Two more for Gold." Card name + photo match the logged bird. **View your … sticker** → Home; **Back to collection** → the album.

## route-1-quests
A working prototype of the **Route 1 · Field guide with quests** flow (built from the Figma "Route 1 · Field guide with quests" page, three bands: onboarding → first quest · log → payoff · coming back → wider game). Same self-contained `index.html` approach as Route 2 — Rubik, brand tokens, real bird photos from the shared folder. **The flow opens with Route 2's first two onboarding screens** (the Welcome splash + the birding-journey question), then continues into the Route-1-specific quest journey. The *Location* screen and the *Log a bird* / map-with-bottom-sheet patterns echo the production app and Route 2.

Flow (14 screens). Screens 1–2 are the shared onboarding lifted from Route 2; 3–14 are the Route 1 screens from the Figma canvas (S1–S11):
1. **Welcome** — the Figma "welcome" frame from the Route 1 page (node 229:152), headline *"Easily identify over 10,000 birds"* (the identify-led variant, no "collect"). Flat 3× export (`images/welcome.png`); tap to continue.
2. **Your birding journey** (Route 2 P3) — single-select (4 levels); button reads *Skip* until you pick, then *Continue*.
3. **Quests built around your location** (S1) — location permission in Route 2's format (same `location-illustration.png`, enlarged to fill the space); the title now hints it's a location request, and the copy frames location as the way quests are tailored to your patch; keeps a "taste of what's waiting" teaser card below (example quest, 0/3).
4. **Your first quest is waiting** (S1.5) — the instant-win quick start. A live quest-primer card (1/3) plus "Seen any of these recently?" — a photo pick-list of common garden birds. **Blue Tit is pre-picked** and pre-credits the quest to **1/3**; tapping more birds is recognised but never credits progress or sightings (honesty rule — only the first recall gives a head start, the rest must be logged in the field). Copy says "remembered", never "spotted". **"Let's go birding" transition:** the extra content (title, pick-list) fades away and the quest-primer card hands off to the map screen's bottom sheet, which starts near-full-height and **shrinks down into place**, revealing the map (the bird list fades up as it settles).
5. **Map home** (S2) — map (CSS, hex sighting-clusters + location dot) with a bottom sheet: the active quest (1/3) + **"Quest Birds Likely Near You"** (Robin, House Sparrow, Blue Tit), each with a small **"+ Log"** button beneath it (→ the log screen). The sheet rests tall enough (`top:44%`) to show all its contents — quest card, bird row, and bottom nav — without scrolling. (The sheet is the landing point of the screen-4 shrink-down transition.)
6. **Your quests** (S3) — the week's quests: Spot 3 garden birds (1/3, active), Find a duck (locked), Spot something new (locked); a Winter waders teaser. The active quest is an **accordion**: tapping it (it no longer jumps to the log screen) expands a **"Likely near you"** list of small bird rows (Robin, House Sparrow, Blue Tit — each with a **+ Log** button → log screen); the standalone "Likely near you" section was folded into here. Resets to collapsed on each visit. Has the **bottom tab bar** (Challenges active) and a **"Find My Next Bird"** CTA above it that returns to the map (5).
7. **Quest detail** (S4) — *(hidden)* Spot 3 common garden birds, 1/3; suggested birds (Blue Tit ticked from recall, Robin + House Sparrow open) labelled "any Common garden bird counts"; a Robin habitat tip; community count.
8. **Log a bird** (S5) — Robin log screen with a **"This logs toward: Spot 3 garden birds · 1 of 3"** chip, detected location, date/time, a count stepper, add-note.
9. **Post-log payoff** (S6) — "Robin logged" (small inline check, no giant tick; content sits clear of the top-right balance). A **unified white header card (soft shadow)** holds the progress **ring** (2/3, "garden birds"), the **time limit** as a prominent bold-amber **live countdown** ("⏳ Quest closes in · 3d 21:14:08", ticking every second), and **ambient social proof** ("👥 14 people completed this quest in the past hour"). Below: the cream **+10 feathers** banner ("Log all 3 to earn +30 more") with a gold **"View ›"** button (whole banner tappable → Your feathers), and a light-blue **"Likely nearby"** card (House Sparrow). Palette is white/cream/light-blue cards rather than grey. A **feather balance** sits top-right, starting at **0** (brand-new user); on arrival a handful of feathers fly subtly from the +10 box up into it and the balance **ticks up 0 → 10** (small pop). Both the balance and the feathers banner open Your feathers; replays each time you land on the screen.
10. **Quest complete** (S7) — *(hidden — see below)* trophy beat; **+30 feathers** (quest completion, now 60); "Find a duck" now unlocked; waders teaser.
11. **Return home** (S8) — *(hidden)* map again, now with a **🪶 60** feather pill; "Good morning — ready to find a duck?"; duck quest (0/1) + Likely near you (Mallard, Tufted Duck, Moorhen).
12. **Your quests — week view** (S9) — *(hidden)* Spot 3 garden birds *Done*; Find a duck (0/1, active); Spot something new (0/1). Challenges tab active.
13. **Your feathers** (S10) — wallet: balance 60, "How you earned them" (per-log + quest breakdown), and "What you can do with feathers": two **affordable rewards** (green cost + a Redeem/Donate button) — *Unlock the next quest early* (40) and *Donate to charity* (50) — then two locked aspirational ones — *Your custom bird guide* (150) and *One month of Birda Pro* (200, with a 60/200 progress bar). Has the **bottom tab bar** (Profile active); the body scrolls with bottom padding so the last card clears the nav. Reached from the payoff's top-right **feather balance** (tap it) or its feathers banner; back returns to the payoff.
14. **Migration event** (S11) — *(hidden)* a limited-time **Winter waders** event: cropped wader hero photo, "Leaving in 21 days", a +60 quest that "Opens 6 July", community count, a "What's a wader?" explainer.

**Currently trimmed to a core flow.** Screens **7, 10, 11, 12, and 14 are hidden** (marked `data-hidden="1"` on the section — kept in the DOM so they can be brought back by just removing that attribute). The live flow is **1–6, 8, 9, + 13 (Your feathers)**. Navigation skips hidden screens, and any link that pointed into one falls through to the next visible screen — so tapping the active quest / *"Find my first bird"* on **Your quests** (was → 7 Quest detail) now goes straight to **Log a bird** (8), and `#7`/`#10`/`#11`/`#12`/`#14` resolve to the next visible screen. Three links were repointed while trimmed (each noted with an HTML comment preserving the original target): the Log screen's *"Cancel"* (was → 7) now → Your quests, the payoff's *"Keep going"* (was → 10) now → map, and Your feathers' *"Back"* (was → 11) now → payoff.

**Persistent feather balance.** A single 🪶 balance pill is overlaid top-right of the phone (one element at the viewport level, so it sits consistently across screens and never blocks content) and **taps through to Your feathers** (13). It reads **0** for the brand-new user, animates **0 → 10** on the payoff (the +10 flying in), then persists at 10. It's shown on the in-app screens and hidden during onboarding (1–4) and on Your feathers itself (redundant there). To make room, the map's decorative menu button was dropped and the quests header reserves right padding.

**Walkthrough panel (same format as Route 3).** On wide screens a context panel sits beside the phone (hidden under 900px) explaining, per screen: the **stage**, the **game mechanics in play** (pills — **CORE** = forms the loop, in blue; supporting layers in light blue), and a snappy **why it works**. Mechanics are mapped from the Discovery board's *"Aligning on routes to explore"* — Route 1's **CORE** mechanics are **Quests** and **Limited-time events**, with **progress meters, near-miss nudges, soft currency (feathers), ambient social proof** (plus **endowed progress / quick win** in onboarding and **currency sinks** on the wallet) as supporting layers. The Map home carries a **🎯 Testable slice** flag (the quest → log → feathers loop). The thesis line frames why the route fits the goal: real-world, findable-now urgency that activates brand-new birders and keeps them coming back.

Honesty rules from the design docs are preserved: nudges use likelihood / habitat tips / community aggregates, never false precision ("a robin is 290m away"); recall pre-credits *quest progress only*, never sightings or feathers.

## route-3-mastery
A working prototype of the **Route 3 · Mastery & identity** flow — *"the recognition loop"* (built from the Figma "Route 3 · Mastery and identity" page — 10 screens, three bands: onboarding → know what you know · log → grow your confidence · track your mastery). The Figma frames are a low-fidelity *Static flow · DRAFT* (emoji placeholders, flat lists); this prototype keeps that flow and copy but **polishes it to Route 2's fidelity** — Rubik, brand tokens, real bird photos from the shared folder, smooth transitions. The species-select and Review Sighting screens are reused from Route 2 / `PEX-43-v2`; the home reuses Route 2's map-home idea (a CSS local-map hero) with a Route 3 **"Your birds" mastery section** layered in. The core idea: **logging is the game** — every log earns XP and levels up your *mastery* of that species (Spotter → Learner → Identifier → Knower → Expert). You're not collecting things; you're becoming a better birder.

A shared **mastery-rank pill** (tier colour scale) and progress bars run through every screen, with a deliberate two-tier bar language: **user/overall XP bars are gold** (thicker, warm amber gradient — the Home level card and the rank-reveal meter) while **per-bird mastery bars are thin and cool** (tier-coloured: blue Learner, grey Spotter, etc.), so "your XP" reads as visually distinct from "this bird's mastery". The single `index.html` carries the picked species name/rank/photo and the mastery copy through Review → Payoff → Species page via `selectSpecies()`.

A **walkthrough context panel** sits beside the phone (hidden below 900px wide). It leads with the route thesis (*"Birda is where you learn to truly know the birds around you… every ID levels up your mastery, Spotter → Expert"*) and then, per screen, names the **game mechanics in play** — mapped from the Discovery FigJam board (the Route 3 row of *"Aligning on routes to explore"*, cross-checked against the full *"Mapping game mechanics"* inventory): **CORE — Levels & XP** (the loop) plus the supporting layers **Endowed Progress, Explicit Progress Meters, Near-Miss Nudges, Streaks (+ Freeze), Variable/Surprise Rewards, Quests/Challenges, Badges, Identity/Profile Customisation, Ambient Social Proof, Geo & Time-Box Splits** — every label taken verbatim from the grid (no invented mechanics), marking CORE vs supporting, with a one-line *why it works* tying each stage to buy-in / habit / identity / retention. The generic, already-exists-in-app screens (Welcome, birding journey, the "What did you see?" species picker) show just the thesis + title; the **Home / map** screen is flagged as the **🎯 Testable slice** — the core log → reward loop to put in front of users first.

**The flow opens with Route 2's first two onboarding screens** (the Welcome splash + the birding-journey question), then continues into the Route-3-specific recognition loop.

Flow (12 screens). Screens 1–2 are the shared onboarding lifted from Route 2; 3–12 are the Route 3 screens (RL S1–S11) from the Figma canvas:
1. **Welcome** (Figma "Welcome" frame) — flat export (`images/figma-reference/welcome.png`), *"Easily identify over 10,000 birds"* (the variant without "collect"); tap to continue.
2. **Your birding journey** (Route 2 P3) — single-select (4 levels); button reads *Skip* until you pick, then *Continue*.
3. **Welcome** (RL S1) — "Build a real skill — one bird at a time"; a Great Tit hero + three value chips (Earn XP · Level up species · Spotter → Expert). **Let's start**.
4. **Calibration picker** (RL S2) — "Which of these have you spotted before?" — a 6-bird photo grid (Robin, Blue Tit, Great Tit, Blackbird, House Sparrow, Wood Pigeon), **multi-select** (Robin / Great Tit / House Sparrow pre-picked). "Pick any — or none." **Done**.
5. **Rank reveal** (RL S3) — *endowed progress*: "You're starting as a **Level 2 Spotter**" (a popped-in badge) with an **XP meter that animates in two beats on arrival** — it fills from zero to full ("Level Two Progress", counting to 40 XP = the head start leveling a brand-new user up to Level 2), then **resets**, relabels to "Level Three Progress", and fills a little further (to 60 XP) with a "+60 XP head start" fading in — credit for the birds you already know. Below, those known birds are listed at Spotter rank under "You've gained a rank in these birds:", with the nudge that the real way to learn is to go and log. **Start logging** / *Look around first* → Home.
6. **Home** (RL S4 + map) — the **full-bleed CSS map + bottom-sheet** shell shared with Routes 1 & 2 (green field, water, road, hex sighting-clusters, location dot, corner search/layers controls — map top kept clean). The bottom sheet leads with a **current-level card** (🐦 Level 2 Spotter · 320 XP · 80 to Level 3 · 🔥 3, with its own progress bar — the level info moved down off the map, replacing the old "Log a bird" hero). Below, a **Your birds** mastery list (Robin *Learner*, Blackbird *Spotter*) — each a vertical row with a tier-coloured progress bar, a **"N logged"** total (prior sightings of that species), and a soft-tinted **Log +1** button at the end of the bar (→ Review Sighting for that bird); tapping the bird's name/photo opens its species mastery page. Then a *"No birds about? Try a quick practice"* row → Sofa practice. 3-item bottom nav (Home · **+** · Profile) inside the sheet — the **+** opens the full species picker.
7. **What did you see?** (RL S5, species select) — reused from Route 2 (search + list: Unidentified, Woodpigeon, Carrion Crow, Robin, Magpie, Blackbird); picking carries the species through the rest of the flow.
8. **Review Sighting + mastery** (RL S6) — the Route 2 / PEX-43-v2 review screen (map pin, photos, notes, sighting details, privacy) with a **mastery progress block** inserted in the Sighting card: *"Robin mastery · 2 more to Identifier"*, a tier bar, and *"This'll be your 6th Robin — logging it gets you closer."* **Save** → payoff.
9. **Post-log payoff** (RL S8 ★) — a deliberately **simple, celebratory** screen: a big gold **⚡ +20 XP** hero (pops in with a confetti burst — no giant green check) over "Logged — Blackbird"; a card showing the **Spotter → Learner** tier jump + progress + "That's your 3rd Blackbird"; a **🔥 4-week streak** line; and a **✨ Surprise unlock** card (a rare *Dawn Glow* profile effect with a tappable **Equip**, twinkling sparkles). **Keep going** / *Done for now* → Home.
10. **Species mastery** (RL S9 ★) — the per-species page: photo + name + current rank, *"Logged 5 times · 3 more to reach Identifier"* with a progress bar, the **5-step mastery ladder** (Spotter ✓ done · Learner current · Identifier / Knower / Expert ahead), and a **"What gives it away"** ID-tips list. **Practice this bird** → Sofa practice.
11. **Profile** (RL S10) — identity, not leaderboards. Reuses **Route 2's profile header** (`images/profile-header.png` — sky cover, avatar, name, following/followers, Edit Profile), then a level-focused body: a **level hero** card (a "LVL 2" emblem + **Spotter** + 3-week streak, with the gold user-XP bar showing **1,240 / 1,500 XP · 260 XP to Level 3 · Scout**); an **ambient social-proof** card (distinct teal treatment + real Tufted Duck photo — "🏆 Local standout · You're Surrey's #1 Tufted Duck expert · ahead of every other birder in your area"); a **Species mastery** distribution (a stacked tier bar + legend — 9 Spotter · 4 Learner · 1 Identifier · 0 Knower · 0 Expert); and a **Badges** grid (earned: First Log, Week Streak, *Dawn Glow* — the payoff drop; in-progress: Thrush Family 2/5 Learner; locked: Raptor Eye, Song ID). 3-item bottom nav.
12. **Sofa practice** (RL S11, secondary) — the no-bird-day drill: "No birds about?", a **❄️ Streak freeze on** note (drills keep the streak alive), and a **Spot-the-difference** card (Blackbird or Carrion Crow?). **Start practice**.

Reached deep-link-style via `index.html#N`; the natural spine is 1→…→9, with the species page (10), profile (11) and practice (12) as branches off Home / the payoff.

### How to use
Open `route-3-mastery/index.html`, `route-1-quests/index.html` (or `route-2-collections/index.html`) in any browser. It's **interactive**: tap the buttons, select options, pick a bird — the flow advances as it would in the app. `← / →` arrow keys jump between screens, and `index.html#5` opens directly on screen 5 (handy for sharing a specific step).

This is a front-end prototype (local state only — no real auth/API). Illustration-heavy screens (the account loader, location map) are approximated; everything else is faithful to the Figma designs.
